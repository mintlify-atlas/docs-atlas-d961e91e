title: Quickstart
description: Go from an empty directory to a running Hummingbird HTTP server in under five minutes.
---

This guide walks you through creating a new Swift package, adding Hummingbird as a dependency, writing a simple HTTP server, and verifying it works with `curl`.

## Prerequisites

<CardGroup cols={2}>
  <Card title="Swift 5.10 or later" icon="swift">
    Download Swift from [swift.org/install](https://www.swift.org/install/). Run `swift --version` to confirm.
  </Card>
  <Card title="macOS 14+ or Linux" icon="terminal">
    Hummingbird supports macOS 14+, iOS 17+, and any Swift-supported Linux distribution.
  </Card>
</CardGroup>

<Note>
  On macOS, Xcode 15 or later ships with a compatible Swift toolchain. On Linux, use the official Swift toolchain from [swift.org](https://www.swift.org/install/).
</Note>

## Build your first server

<Steps>

<Step title="Create a new Swift package">

Create an executable Swift package in a new directory:

```bash
mkdir HelloServer && cd HelloServer
swift package init --type executable
```

This produces the following layout:

```
HelloServer/
├── Package.swift
└── Sources/
    └── main.swift
```

</Step>

<Step title="Add Hummingbird to Package.swift">

Open `Package.swift` and replace its contents with the following:

```swift Package.swift
// swift-tools-version:5.10
import PackageDescription

let package = Package(
    name: "HelloServer",
    platforms: [.macOS(.v14)],
    dependencies: [
        .package(url: "https://github.com/hummingbird-project/hummingbird.git", from: "2.0.0")
    ],
    targets: [
        .executableTarget(
            name: "HelloServer",
            dependencies: [
                .product(name: "Hummingbird", package: "hummingbird"),
            ]
        ),
    ]
)
```

<Tip>
  You can add the dependency from the command line without editing `Package.swift` manually. Replace `HelloServer` with the name of your target:

  ```bash
  swift package add-dependency https://github.com/hummingbird-project/hummingbird.git --from 2.0.0
  swift package add-target-dependency Hummingbird HelloServer --package hummingbird
  ```
</Tip>

</Step>

<Step title="Write the server">

Replace the contents of `Sources/main.swift` with:

```swift Sources/main.swift
import Hummingbird

// Create a router and register a GET /hello route
let router = Router()
router.get("hello") { request, _ -> String in
    return "Hello, world!"
}

// Create the application with the router and bind address
let app = Application(
    router: router,
    configuration: .init(address: .hostname("127.0.0.1", port: 8080))
)

// Run the server — gracefully shuts down on SIGINT/SIGTERM
try await app.runService()
```

`Application` conforms to `Service` from swift-service-lifecycle. `runService()` wraps the app in a `ServiceGroup` that listens for `SIGINT` and `SIGTERM` and performs a clean shutdown when either signal is received.

</Step>

<Step title="Add a route with a path parameter">

Route parameters are declared with a `:` prefix in the path pattern. The router extracts the matched segment and makes it available through `context.parameters`.

Add the following route below the `router.get("hello")` call:

```swift
router.get("hello/:name") { request, context -> String in
    // require() throws HTTPError(.badRequest) if the parameter is missing
    let name = try context.parameters.require("name")
    return "Hello, \(name)!"
}
```

Your full `main.swift` should now look like this:

```swift Sources/main.swift
import Hummingbird

let router = Router()

router.get("hello") { request, _ -> String in
    return "Hello, world!"
}

router.get("hello/:name") { request, context -> String in
    let name = try context.parameters.require("name")
    return "Hello, \(name)!"
}

let app = Application(
    router: router,
    configuration: .init(address: .hostname("127.0.0.1", port: 8080))
)
try await app.runService()
```

<Info>
  `context.parameters.require(_:)` returns the parameter as a `String` and throws an `HTTPError(.badRequest)` if the key is absent. Use `context.parameters.get(_:)` if you prefer an optional return value instead of a thrown error.
</Info>

</Step>

<Step title="Run the server">

Start the server with:

```bash
swift run
```

Swift will resolve and compile dependencies on the first run, which takes a moment. Once the server is ready you will see log output similar to:

```
info Hummingbird : Server started and listening on 127.0.0.1:8080
```

</Step>

<Step title="Test with curl">

Open a second terminal and send requests to both routes:

```bash
curl http://127.0.0.1:8080/hello
```

```
Hello, world!
```

```bash
curl http://127.0.0.1:8080/hello/Swift
```

```
Hello, Swift!
```

Press `Ctrl+C` in the server terminal to stop the process. Hummingbird handles `SIGINT` and drains in-flight requests before exiting.

</Step>

</Steps>

## What you built

| Route | Handler | Response |
|---|---|---|
| `GET /hello` | Returns a static string | `Hello, world!` |
| `GET /hello/:name` | Reads the `:name` parameter | `Hello, <name>!` |
