## Why Hummingbird?

Hummingbird is built for Swift developers who want a performant, composable HTTP framework without unnecessary complexity. It embraces Swift Concurrency with full async/await support and integrates seamlessly with the broader Swift server ecosystem.

<CardGroup cols={2}>
  <Card title="High performance routing" icon="bolt">
    Trie-based router with wildcard and named parameter support for fast path matching
  </Card>
  <Card title="Composable middleware" icon="layer-group">
    Chain middleware to handle authentication, logging, CORS, tracing, and more
  </Card>
  <Card title="Type-safe request context" icon="shield-check">
    Pass typed data through the request pipeline without dictionaries or type casting
  </Card>
  <Card title="Minimal dependencies" icon="feather">
    Only depends on Apple's open-source Swift packages — no heavy third-party libraries
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Add Hummingbird to your package">
    Add the dependency to your `Package.swift` file and run `swift package resolve`.

    ```swift Package.swift
    dependencies: [
        .package(url: "https://github.com/hummingbird-project/hummingbird.git", from: "2.0.0")
    ],
    targets: [
        .executableTarget(
            name: "MyApp",
            dependencies: [
                .product(name: "Hummingbird", package: "hummingbird"),
            ]
        )
    ]
    ```
  </Step>
  <Step title="Create your first route">
    Import Hummingbird and define a router with at least one route handler.

    ```swift main.swift
    import Hummingbird

    let router = Router()
    router.get("hello") { request, _ -> String in
        return "Hello, world!"
    }
    ```
  </Step>
  <Step title="Run your application">
    Wrap the router in an `Application` and call `runService()` to start the server.

    ```swift main.swift
    let app = Application(
        router: router,
        configuration: .init(address: .hostname("127.0.0.1", port: 8080))
    )
    try await app.runService()
    ```
  </Step>
</Steps>
